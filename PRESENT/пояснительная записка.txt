В коде
функции:
game_process:
keyPressEvent - обрабатывает нажатие клавиш
kb_call - вызывает форму для изменения назначений клавиш
set_settings - задает параметры игры, которые ввел пользователь
	Если нет ключа - принимает значения из окна settings у DialWidget и записывает их в переменные.
	Если задан ключ - берет все даннные из ключа и так же записывает значения в пременные
create_map - создает игровую карту при мпомощи переменных записанных в функции set_settings(Заполняет массив классами Cell)
show_map - показывает игровую карту и игрока пользователю после создания карты в create_map
convert_to_36 - конвертирует из 10-ричной в 36-ричную систему счисления 
	(Нужен для сокращения длинного ключа генерации путем конвертации в 36 ричную систему счисления)
create_key - создает ключ генерации принимая игровую карту 
	(каждую клетку карты переводит в 1 если лава и в 0 если пусто)
create_map_by_key - создает карту принимая ключ генерации
	(создает массив из классов Cell если 1 это лавва. если 0 это пустая клетка)
Функции управления - передвижение и взаимодействие с картой
	(вверх, вниз, вправо, влево, проверить наличие руды под игроком, скопать путь на нижний этаж, подняться, спуститься) 
cheating - проеверяет цепочку движений на савподение по konami-code
game_ending - заканчивает игру и запрещает движение
lava_burn - проверяет попал ли игрок в лаву
return_with_ore - при возвращении к кораблю проверяет есть ли у игрока руда
keyboard:
user_kb - открывает файл с пользовательскими настройками и применяет их
write_user_settings - вводит файл измененные пользовательские настройки
show_letters - показывает пользователю значение клавиш
apply - сохраняет пользовательские настройки
"btn"_change - принимают изменение клавиш
DataBase:
show_all - выводит все рекорды в таблицу
add_new_player - добовляет нового игрока
add_new_key - добовляет нового игрока
add_new_record - добовляет новый рекорд
change_record - изменяет рекорд
save_result - вызывает диалоговое окно где пользователь решает сохранить ли результат
cancel - закрывает окно


классы:
Cell - класс отдельной клетки поля, хранит данные о поле(Например: type='lava', way_down=False, way_up=True)
Game - класс главного окна
KboardForm - класс изменения клавиш
Winning - класс для сохранения результатов
